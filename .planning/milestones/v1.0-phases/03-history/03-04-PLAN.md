## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| historyStore → IPC | All invoke calls use exact command names; no dynamic command string construction |
| ConversationRow snippet display | FTS5 snippet() output rendered as plain text, NOT with @html — prevents XSS via stored message content |
| surfaceStore.setSurface | Navigation uses the typed Surface union — 'chat' literal only, not derived from stored content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-16 | Information Disclosure | ConversationRow renders message content as snippet | mitigate | Snippet rendered as plain text (no @html); FTS5 markers like <b>term</b> appear as literal text, not HTML — no injection surface |
| T-03-17 | Tampering | historyStore.search sends user-typed query to FTS5 via invoke | mitigate | FTS5 MATCH uses parameterized bind in Rust (Plan B T-03-04); frontend just passes string through invoke — no SQL construction in frontend |
| T-03-18 | Denial of Service | SearchBar fires invoke on every debounce tick with no debounce cancellation on unmount | mitigate | $effect cleanup returns clearTimeout(debounceTimer) to prevent post-unmount invocations |
| T-03-19 | Tampering | ConversationRow delete confirmation uses frontend-only state — no backend confirmation | accept | IPC call is the actual delete gate; inline confirmation is UX only and cannot be bypassed without calling history_delete which enforces main window |
| T-03-20 | Information Disclosure | historyStore.error may contain IPC error message string | accept | normalizeIpcError extracts user-facing message string only; no raw Rust panic or stack trace crosses IPC |
| T-03-SC | Tampering | npm/pnpm install of new frontend deps | accept | No new npm dependencies added in this plan — all components use existing Svelte 5 + @tauri-apps/api/core already in package.json |
</threat_model>
