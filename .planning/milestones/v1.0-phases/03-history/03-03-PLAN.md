## Trust Boundaries

| Boundary                    | Description                                                                                                                                     |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| chat_send parameter         | conversation_id: Option<String> accepted from frontend — used only to look up existing conversation, never accepted as a title or content value |
| Spawned task storage access | state re-acquired via app*handle.state::<T>() inside spawn; no State<'*> lifetime crossing thread boundary                                      |
| Title generation            | Auto-generated from first user message content — D-03: title is backend-owned, never accepted from IPC                                          |

## STRIDE Threat Register

| Threat ID | Category               | Component                                                                            | Disposition | Mitigation Plan                                                                                                                                   |
| --------- | ---------------------- | ------------------------------------------------------------------------------------ | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-03-11   | Tampering              | conversation_id accepted from frontend could reference another user's conversation   | accept      | Single-user desktop app; no multi-tenancy; conversation_id is used to append messages to existing conversation                                    |
| T-03-12   | Information Disclosure | title_from_messages includes user prompt content as conversation title               | accept      | Title stays backend-owned (D-03); never returned to frontend directly; only via history_list which is restricted to main window                   |
| T-03-13   | Denial of Service      | storage write failure in spawned task blocks streaming                               | mitigate    | Storage errors in spawned task are eprintln! non-fatal; streaming continues regardless of storage write outcome                                   |
| T-03-14   | Tampering              | assistant text accumulation could be exploited if provider sends adversarial content | accept      | Content stored verbatim per D-04; egress redaction is Phase 4; no execution of stored content                                                     |
| T-03-15   | Tampering              | conversation_id injected by malicious renderer to corrupt another conversation       | accept      | Same mitigation as T-03-11 — single user, no secrets crossing IPC, worst case: extra messages in wrong conversation (user recoverable via delete) |

</threat_model>
