<threat_model>

## Trust Boundaries

| Boundary                | Description                                                                                                                |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| renderer → IPC          | Frontend invoke() calls cross into backend command handlers; only commands granted in capabilities/main.json are reachable |
| backend → SQLite        | ShellPreferenceStore is the sole writer/reader of shell_preferences; renderer never issues raw SQL                         |
| app handle → filesystem | setup() resolves app_data_dir and creates the SQLite file under the OS-owned per-app data directory                        |

## STRIDE Threat Register

| Threat ID | Category               | Component                           | Disposition | Mitigation Plan                                                                                                                                             |
| --------- | ---------------------- | ----------------------------------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-01-01   | Elevation of Privilege | new app-shell.toml permissions      | mitigate    | Permissions allow ONLY get_active_surface and set_active_surface scoped to the "main" window via the existing capability; no wildcard command grant         |
| T-01-02   | Tampering              | set_active_surface persistence path | mitigate    | Surface is a closed enum deserialized by serde; invalid values rejected before any SQL; assert_main_window enforces window label as defense-in-depth        |
| T-01-03   | Information Disclosure | SQLite file location                | accept      | DB lives under OS per-app data dir; contains only a non-sensitive surface name (chat/history/settings/artifacts), no secrets or prompt content              |
| T-01-04   | Denial of Service      | migrations on open()                | mitigate    | run_migrations is idempotent and wraps each migration in a savepoint; a failed migration rolls back and surfaces as a setup error rather than a corrupt DB  |
| T-01-SC   | Tampering              | npm/cargo installs                  | accept      | No new package-manager installs in this plan; all dependencies (tauri 2, rusqlite, serde, thiserror) already present in Cargo.toml and reviewed in Phase 01 |

</threat_model>
