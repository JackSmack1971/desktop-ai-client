> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-authored Svelte scoped styles |

## Spacing Scale

Declared values extracted from existing components (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (SurfaceRail gap: 4px) |
| sm | 8px | Button padding, row gaps (ChatInput gap: 8px; confirm-actions gap: 12px rounds to sm) |
| md | 16px | Default element spacing, horizontal padding (surface-header: 16px 24px) |
| lg | 24px | Section padding (surface-header horizontal: 24px) |
| xl | 32px | Layout gaps (not yet used — reserved for section breaks) |

| 3xl | 64px | Page-level spacing (reserved) |

Exceptions:
- Rail button: 48x48px (matches 2xl token — satisfies 44px minimum touch target with margin)
- Delete confirmation inline panel: 12px padding (sits between sm and md — use md=16px on new panels)
- Focus ring: `outline: 2px solid #4a9eff; outline-offset: 2px` — declared everywhere, mandatory on all interactive elements

## Phase 4 Component Inventory

Only one Svelte component is in scope for Phase 4 frontend work.

### `SettingsSurface.svelte` (replace scaffold)

**Purpose:** Credential management only. API key write, status display, and clear. No model selection, no theme, no retention config (deferred per D-11).

**Layout structure:**

```
.surface.settings-surface          (flex column, height: 100%, width: 100%)
  .surface-header                  (padding: 16px 24px, border-bottom: 1px solid #2a2a2a)
    h1.surface-title               (16px/600, color: #e0e0e0) — "Settings"
  .surface-body                    (flex: 1 1 auto, overflow: auto, padding: 24px)
    section.credentials-section    (max-width: 480px, display: flex, flex-direction: column, gap: 24px)
      h2.section-heading           (14px/600, color: #e0e0e0) — "API Credentials"
      .credential-card             (background: #242424, border: 1px solid #2a2a2a, border-radius: 8px, padding: 16px)
        .provider-label            (12px/400, color: #888) — "OpenRouter"
        .status-row                (display: flex, align-items: center, gap: 8px)
          .status-indicator        (see Status Indicator spec below)
        form.key-form              (display: flex, flex-direction: column, gap: 8px — only shown when status is MISSING or after user clicks "Replace")
          input[type="password"]   (see API Key Input spec below)
          .form-actions            (display: flex, gap: 8px, justify-content: flex-end)
            button.btn-save        (see Save Button spec below)
        button.btn-clear           (see Clear Button spec below — only shown when status is CONFIGURED)
      .error-region                (role="alert", see Error Region spec below)
```

**Status Indicator:**

- When `status === 'CONFIGURED'`: dot `#4a9eff` (8px circle) + text "API key configured" in `#e0e0e0` 14px/400
- When `status === 'MISSING'`: dot `#888` (8px circle) + text "No API key set" in `#888` 14px/400
- When `loading === true`: dot animated pulse (opacity 0.4 → 1.0 cycle, 1.2s) + text "Checking…" in `#888` 14px/400

**API Key Input (`input[type="password"]`):**

Identical visual spec to `.chat-textarea` / `.search-input` pattern:
- Height: 40px
- Background: `#1a1a1a`
- Border: `1px solid #2a2a2a`
- Border-radius: 6px
- Padding: 0 12px
- Color: `#e0e0e0`
- Font-size: 14px
- Placeholder text: "Paste your API key" — color `#666`
- Focus: `border-color: #4a9eff; box-shadow: 0 0 0 2px rgba(74,158,255,0.25)`
- `autocomplete="off"` — prevent browser autofill of credential fields
- `aria-label="API key for OpenRouter"`

Security invariant: `type="password"` is mandatory. The input value MUST be cleared immediately after the IPC call completes (success or failure). The component must never render the key back from the backend.

**Save Button (`.btn-save`):**

- Text: "Save API Key" (primary CTA — see Copywriting Contract)
- Background: `#4a9eff`
- Color: `#fff`
- Font-size: 14px; font-weight: 500
- Padding: 8px 16px
- Border-radius: 6px
- Border: none
- Disabled state: `opacity: 0.6; cursor: not-allowed` (while loading or when input is empty)
- Hover: `background: #3a8eef`
- `aria-label="Save API key for OpenRouter"`
- `aria-busy={loading}` when the IPC call is in flight

**Clear Button (`.btn-clear`):**

- Text: "Remove key" (destructive secondary — see Copywriting Contract)
- Background: transparent
- Color: `#888`
- Font-size: 14px; font-weight: 400
- Padding: 6px 0
- Border: none
- Hover: `color: #cc4444`
- Shown only when `status === 'CONFIGURED'`
- On click: opens inline confirmation (inline, not modal — consistent with ConversationRow delete pattern)

**Clear Confirmation (inline, not modal):**

Pattern follows ConversationRow `.delete-confirm`:
- Container: `background: #2a1a1a; border-top: 1px solid #7a1e1e; padding: 12px 16px; border-radius: 0 0 6px 6px`
- Heading (14px/400, `#e0e0e0`): "Remove API key?"
- Body (14px/400, `#888`): "This will clear the key from the OS keychain. You will need to re-enter it to use AI features."
- Actions: "Confirm remove" button (background: `#2a1a1a`, border: `1px solid #7a1e1e`, color: `#cc4444`, 14px, 6px 12px, border-radius: 4px) + "Cancel" link-button (`#888`, 14px, transparent background)
- `role="alertdialog"`, `aria-label="Confirm key removal"`, `aria-modal="true"` — matches ConversationRow ARIA pattern
- Escape key cancels and returns focus to the "Remove key" button

**Error Region (`.error-region`):**

- Position: below the credential card
- `role="alert"` (live region — announces immediately on content change)
- Text: 14px/400, color `#cc4444`
- Margin-top: 8px
- Empty when no error — do not render the element at all when error is null (prevents empty live region announcements)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Save API Key" |
| Status — configured | "API key configured" |
| Status — missing | "No API key set" |
| Status — loading | "Checking…" |
| Empty state heading | "No API key set" |
| Empty state body | "Paste your OpenRouter API key below to enable AI features." |
| Error — store failure | "Failed to save key. Check your OS keychain access and try again." |
| Error — clear failure | "Failed to remove key. Check your OS keychain access and try again." |
| Error — status check fail | "Could not check credential status. Try restarting the app." |
| Destructive action label | "Remove key" (button triggers confirmation) |
| Destructive confirmation heading | "Remove API key?" |
| Destructive confirmation body | "This will clear the key from the OS keychain. You will need to re-enter it to use AI features." |
| Destructive confirm CTA | "Confirm remove" |
| Destructive cancel | "Cancel" |
| Section heading | "API Credentials" |
| Provider label | "OpenRouter" |
| Input placeholder | "Paste your API key" |

Copy rules:
- Never use "delete" for credential removal — use "remove" or "clear" (delete implies permanent data loss; removal from keychain is recoverable by re-entry)
- Never display partial key, masked key, or any derivative of the key value
- Status copy is always one of three states: "API key configured" / "No API key set" / "Checking…" — no other variants
- Error copy always ends with a recovery action the user can take (not dead-end errors)

## Interaction Contract

### State Machine

```
IDLE (status unknown)
  → on mount: call privacy_get_credential_status → LOADING
LOADING
  → success, CONFIGURED: show status badge + "Remove key" button; hide form
  → success, MISSING: show status badge + key entry form
  → error: show error region; show key entry form (allow retry)
CONFIGURED
  → click "Remove key": show inline confirmation
  → confirm: call privacy_clear_provider_key → LOADING
  → cancel: hide confirmation, return focus to "Remove key" button
MISSING
  → user types in input: enable "Save API Key" button
  → click "Save API Key" (valid input): call privacy_set_provider_key → SAVING
SAVING
  → success: clear input value, transition to CONFIGURED state
  → error: clear input value, show error region, stay in MISSING state
CLEARING
  → success: hide confirmation, transition to MISSING state
  → error: hide confirmation, show error region, stay in CONFIGURED state
```

### Security Interaction Invariants

- The API key input field value is cleared from `$state` immediately after the IPC call resolves (both success and failure paths)
- The "Save API Key" button is disabled when the input field is empty or whitespace-only
- No IPC response ever populates the input field — status is always binary (CONFIGURED/MISSING)
- If `status === 'CONFIGURED'`, the key entry form is hidden (not just disabled); revealing it requires clicking a "Replace key" affordance (optional UX polish, not required for Phase 4)

### Keyboard Contract

- Tab order: section heading → status indicator (non-interactive, skip) → API key input (when visible) → "Save API Key" button → "Remove key" button (when visible)
- Inside confirmation: "Confirm remove" → "Cancel"; Escape dismisses
- All interactive elements have `focus-visible` outlines: `outline: 2px solid #4a9eff; outline-offset: 2px`
- After confirmation cancel: focus returns to the "Remove key" button

### ARIA Contract

| Element | ARIA Role / Attribute |
|---------|----------------------|
| Settings region | `role="region" aria-label="Settings"` (matches existing scaffold) |
| Credentials section | `role="group" aria-labelledby="credentials-heading"` |
| Status indicator | `aria-live="polite"` on the status row — announces status changes |
| API key input | `aria-label="API key for OpenRouter"` |
| Save button | `aria-label="Save API key for OpenRouter"` + `aria-busy={loading}` |
| Remove button | `aria-label="Remove API key for OpenRouter"` |
| Confirmation dialog | `role="alertdialog" aria-label="Confirm key removal" aria-modal="true"` |
| Error region | `role="alert"` — only rendered when error is non-null |
| Loading indicator | `aria-live="polite"` on status dot (covers LOADING state) |
