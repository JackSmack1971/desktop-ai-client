Establish the backend artifact contract for Phase 5: sanitize and wrap generated content, persist artifact rows in SQLite, extend chat streaming to emit `ArtifactReady` after completion, and add backend retrieval/dismiss commands so the frontend can render only sanitized `srcdoc` content.

Purpose: make artifact previews a backend-owned feature with typed IPC and durable storage before any surface chrome relies on them.
Output: artifact sandbox helper, artifacts table/store, chat-event extension, and artifact IPC commands wired into the backend command surface.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → chat IPC | Untrusted frontend can request message sending, but it must never control sandbox policy or supply executable artifact behavior |
| backend → artifact storage | Raw generated content stays backend-owned until sanitized and wrapped for preview |
| backend → frontend preview | Frontend receives sanitized `srcdoc` only; no unsanitized artifact source crosses IPC |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | artifact preview IPC | mitigate | sanitize in Rust, re-wrap with strict CSP, and never return raw source to the renderer |
| T-05-02 | Tampering | attachment resolution path | mitigate | resolve tokens backend-side only; do not accept raw frontend file paths |
| T-05-SC | Tampering | artifact command registration | mitigate | register only explicit artifact commands in the handler and capability file |
</threat_model>
