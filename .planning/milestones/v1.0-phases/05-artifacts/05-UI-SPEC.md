## Spacing Scale

Declared values (must be multiples of 4). Derived from measurements across `AppShell.svelte`, `ChatSurface.svelte`, `HistorySurface.svelte`, `ArtifactsSurface.svelte` scaffold, and `SurfaceRail.svelte`.

| Token | Value | Usage                                                                  |
| ----- | ----- | ---------------------------------------------------------------------- |
| xs    | 4px   | Rail button gap, icon-label gap within rail buttons                    |
| sm    | 8px   | Rail vertical padding, surface body padding-y, status region padding   |
| md    | 16px  | Surface header padding-y, surface body padding-x, error banner padding |
| lg    | 24px  | Surface header padding-x, search area padding-x                        |
| xl    | 32px  | Major layout breaks (reserved, not yet in use)                         |
| 2xl   | 48px  | Page-level vertical spacing (reserved)                                 |
| 3xl   | 64px  | Reserved for future full-page layouts                                  |

Exceptions:

- Rail button touch target: 48px × 48px (accessibility minimum for icon-only interactive elements; exceeds the 8-point scale by design)
- Rail width: 56px (fixed; inherits from Phase 1 AppShell layout; do not change)
- Artifact chrome toolbar height: 48px (matches rail button touch target; provides visual alignment)

**Source:** Measured from existing scoped CSS. Tokens are not formally declared; they are inline values replicated consistently across surface files.

## Component Inventory

These are the specific UI components Phase 5 must build or extend. Each inherits the design tokens above.

### ArtifactsSurface.svelte (scaffold → full implementation)

Layout: identical structure to `ChatSurface.svelte` and `HistorySurface.svelte`.

```
┌─────────────────────────────────────────────────────┐
│ surface-header (16px/24px padding, border-bottom)    │
│   h1.surface-title "Artifacts"  [content-type badge] │
│   [toolbar: Reload | Stop buttons — right-aligned]   │
├─────────────────────────────────────────────────────┤
│ surface-body (flex: 1 1 auto, overflow: hidden)      │
│   ┌─────────────────────────────────────────────────┤
│   │ iframe.artifact-frame (flex: 1 1 auto)           │
│   │   srcdoc = sanitized content from backend        │
│   │   sandbox="allow-same-origin"                    │
│   │   (or more restrictive — no allow-scripts)       │
│   └─────────────────────────────────────────────────┤
│   [error-state panel — shown when iframe is empty]   │
└─────────────────────────────────────────────────────┘
```

States:

- `idle`: Show empty state (no artifact loaded). Surface header shows only "Artifacts" title.
- `loading`: Artifact fetch in progress. Toolbar buttons disabled. Show loading indicator in surface-body.
- `ready`: iframe shows sanitized srcdoc. Toolbar shows Reload + Stop buttons (Stop is enabled).
- `dismissed`: iframe srcdoc set to empty / about:blank. Toolbar shows Reload button only (Stop disabled).
- `error`: Backend fetch failed or CSP cannot be applied. Show error panel in surface-body. iframe remains empty.

### Artifact Chrome Toolbar (inside surface-header, right-aligned)

| Element       | Copy     | State visibility                                                       |
| ------------- | -------- | ---------------------------------------------------------------------- |
| Reload button | "Reload" | Visible in `ready`, `dismissed`, `error`                               |
| Stop button   | "Stop"   | Visible and enabled in `ready`; hidden in `idle`, `dismissed`, `error` |

- Both buttons are outside the sandboxed iframe (host chrome — cannot be blocked by artifact content)
- Reload: triggers `artifact_get` IPC call, replaces iframe srcdoc
- Stop: sets iframe srcdoc to empty, aborts in-flight IPC, transitions to `dismissed` state
- Touch target: 48px × 48px minimum (matches rail button standard)
- Focus-visible outline: 2px solid #4a9eff, 2px offset (inherited from SurfaceRail pattern)

### Content-Type Badge (inside surface-header, beside "Artifacts" title)

Displayed when an artifact is loaded. Shows the content type of the active artifact.

| Content Type | Badge Label                                |
| ------------ | ------------------------------------------ |
| HTML         | "HTML"                                     |
| SVG          | "SVG"                                      |
| Plain Text   | "Text"                                     |
| Code         | "Code · {language}" (e.g. "Code · Python") |

Badge styling: 13px/400, background #2a2a2a, color #4a9eff (HTML) or #888 (SVG, Text, Code), border-radius 4px, padding 2px 8px, margin-left 8px.

### Artifact Indicator in ChatSurface (new element in existing surface)

When an `ArtifactReady` event arrives, Chat surface shows a minimal indicator after the assistant message:

```
┌─────────────────────────────────────────────────────────┐
│ [Artifact ready — View artifact →]                       │
│  13px/400, color: #4a9eff, cursor: pointer               │
│  Clicking navigates to Artifacts surface                  │
└─────────────────────────────────────────────────────────┘
```

This is not a separate component — it is a conditional block inside `ChatSurface.svelte` rendered beneath the triggering assistant message. It uses `surfaceStore.setSurface('artifacts')` to navigate.

## Interaction Contracts

### Keyboard Navigation

All controls outside the iframe must be keyboard accessible per ARTF-03.

| Interaction                      | Key             | Behavior                                                                            |
| -------------------------------- | --------------- | ----------------------------------------------------------------------------------- |
| Reach toolbar                    | Tab             | Focus moves from rail to surface-header toolbar after `main-content` receives focus |
| Activate Reload                  | Enter / Space   | Triggers artifact_get IPC, replaces srcdoc                                          |
| Activate Stop                    | Enter / Space   | Empties srcdoc, aborts IPC, transitions to dismissed                                |
| Navigate between toolbar buttons | Tab / Shift+Tab | Standard tab order within toolbar                                                   |
| Focus return after stop          | Auto            | Focus stays on Stop button position; button becomes hidden; focus moves to Reload   |

### Screen Reader Contracts

- `ArtifactsSurface` root div: `role="region"` `aria-label="Artifacts"` (consistent with all other surfaces)
- iframe: `title="Artifact preview"` (required for screen reader identification)
- Toolbar: `role="toolbar"` `aria-label="Artifact controls"`
- Reload button: `aria-label="Reload artifact"` (not just "Reload")
- Stop button: `aria-label="Stop artifact preview"` (not just "Stop")
- Content-type badge: `aria-label="{content type} artifact"` on the surface heading region, or visually alongside via `aria-describedby`
- Loading indicator: `role="status"` `aria-live="polite"` with copy "Loading artifact…"
- Error state: `role="alert"` `aria-live="assertive"` (matches pattern from `ChatSurface.svelte`)
- Artifact indicator in Chat: `role="status"` `aria-live="polite"` with copy "Artifact ready"

### Fail-Closed Guard

Per D-03 and the specifics from CONTEXT.md: if sanitized srcdoc is absent or a fetch error occurs, the iframe must NOT render fallback content. The error panel must replace the iframe view, not fall back silently.

Implementation: `{#if artifactState.state === 'error'}` block replaces the iframe. The iframe element is removed from the DOM in error state (not hidden with CSS), so screen readers do not encounter an empty, unlabeled frame.

## Iframe Sandbox Contract

This section is normative for the executor. Deviations require explicit sign-off.

```html
<iframe
	title="Artifact preview"
	class="artifact-frame"
	srcdoc="{artifactPreview}"
	sandbox="allow-same-origin"
	role="document"
></iframe>
```

- `allow-scripts` must NOT be in the sandbox attribute for Phase 5 (per D-03, D-04)
- `allow-forms` must NOT be in the sandbox attribute
- `allow-popups` must NOT be in the sandbox attribute
- `allow-top-navigation` must NOT be in the sandbox attribute
- `allow-same-origin` may be included if needed for srcdoc same-origin access; evaluate at implementation time. If it can be omitted without breaking srcdoc rendering, omit it.
- The CSP meta tag inside the srcdoc wrapper (generated by `security::artifact_sandbox`) must include: `script-src 'none'; connect-src 'none'; form-action 'none'; object-src 'none'`
- If `artifactPreview` is empty string, null, or undefined, the iframe must NOT be rendered. Show the `idle` or `error` state panel instead.

## Accessibility Release Gate (ARTF-03)

The following are explicit pass/fail criteria for the executor and auditor:

- [ ] Keyboard path: user can reach Reload and Stop buttons without a mouse
- [ ] Focus visible: all interactive elements in artifact chrome show a 2px solid #4a9eff focus ring

- [ ] Screen reader: toolbar has `role="toolbar"` and labelled buttons
- [ ] Screen reader: loading and error states use appropriate live regions
- [ ] Screen reader: artifact indicator in Chat uses `aria-live="polite"`
- [ ] Stop and Reload controls are in host chrome, not inside the iframe
